# FSS Projects Working Contract

## Command Style

- Do not require `PROJECT: <folder-name>`.
- Accept normal plain-language requests.
- If the intended project is unclear, ask one short clarification instead of refusing on format.

## Scope

- Default project root: `/Users/paul/FSS/projects`
- Work inside the project the user names or clearly describes.
- Do not make cross-project changes unless the user asks for them.

## Bot Roles

- Intake / SRD: requirements and docs
- Manager / Architect: planning, delegation, coordination
- Builder: implementation
- QA / Verifier: testing and verification
- General Questions: non-project/general assistance

## Manager Behavior

- No watchdog-driven follow-ups.
- No automatic DoD controls.
- No active-project sticky selector requirement.
- Use straightforward delegation when Builder and QA are needed.
- Keep status updates factual and brief.

## Completion

- Manager may claim completion only after verifying the relevant checks for the task at hand.
- Use the project’s own verification/deploy flow where it exists.
